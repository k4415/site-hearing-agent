# Sheet Templates

クライアントに **1 メッセージで一括送信** するシートの実テンプレート。コピペで使える文面集。

## ⓪ キックオフ 3 問（対話形式・1 問ずつ）

これだけは対話で。1 問ずつ自然に投げる。

```
こんにちは、サイト制作のヒアリングを担当する site-hearing-agent です。
まず簡単に 3 つだけ伺ってから本題に進みますね。

クライアント企業名（または案件名）を教えていただけますか？
```

クライアント回答後:

```
ありがとうございます。
サイトの種類は LP（1 枚もの・キャンペーン用）と、サービス/SaaS の紹介サイト、どちらに近いですか？
```

回答後:

```
ありがとうございます。
最後に、この案件のオーナー（最終判断する人）はどなたですか？（ご自身なら「自分」で OK です）
```

3 つ揃った時点で:
1. 裏で `aachat session run site-strategy-orchestrator.<owner>` を呼んで case doc 作成依頼（非同期）
2. クライアントには「ありがとうございます。それでは事前資料の確認に進みますね」と続けて、下記 ① を一括送信

---

## ① 事前共有資料リクエスト（一括メッセージ）

```
ヒアリングを始める前に、もし手元にある資料があれば URL を共有いただけると、
後の戦略設計とリサーチの精度が大きく上がります。
（全部揃わなくて OK です。「ある分だけ」共有してください）

▼ 事業を理解するための資料
1. 既存サイト URL（自社の現サイト・LP・サービスページ）:
2. 商品/サービスの紹介ページ URL（カタログや料金表でも可）:
3. 過去の提案資料・営業資料の URL（あれば）:
4. お客様の声・レビュー・事例の URL（自社サイト掲載 or 外部レビュー）:

▼ 競合・参考を理解するための資料
5. 直接の競合だと思うサイト URL（2〜3 つ）:
6. 「このサイト好き」と思う参考サイト URL（2〜3 つ）:
7. 「これにはしたくない」サイト URL（あれば）:

▼ 追加情報（任意）
8. 業界レポート・調査資料の URL（あれば）:
9. その他、見ておいてほしい情報の URL:

ファイルを直接送れる場合は aachat にアタッチでも構いません。
```

回答後:
- 各 URL を case doc の `pre_shared_assets` に追記依頼（orchestrator に session send）
- 「ありがとうございます。続けて素材の確認に進みますね」と返して ② へ

## ③ コアシート 10 問（1 メッセージで一括送信）

```
それでは本ヒアリングです。まず最重要の 10 問をまとめて送ります。
順番通りでなくて OK、書ける項目から書いてください。1〜2 文ずつで十分です。

【コアシート（10 問）】

事業の現在地
Q1. このサービスを一言で言うと？（社内の人に説明するように）

JTBD（決定打）
Q2. 最近契約してくれたお客さんは、契約前は何で困っていましたか？
Q3. そのお客さんが「これだ」と決めた瞬間、何が決め手でしたか？
Q4. もしウチがなかったら、お客さんは何で代用しますか？（具体名で）

ペイン / ゲイン
Q5. お客さんがよく口にする愚痴・困りごとは？（生の言葉のまま）
Q6. 契約後に「これは助かった」と言われたことは？

サイトの役割
Q7. このサイトに来た人に、最終的にしてほしい 1 つの行動は？
Q8. 月にどれくらい問い合わせ / CV があれば「成功」と言えますか？

オファー
Q9. 価格はどう設定していますか？（プラン構造・レンジ）

ブランド
Q10. ブランドの「らしさ」を一言で言うと？
```

回答後:
- 発話ログに `[<client-name>] "..."` 形式で原文を記録
- 構造化要約セクションを更新
- coverage を評価
- 「ありがとうございます。続けて詳細シートに進みますね」と返して ④ へ

## ④ 詳細シート 20 問（1 メッセージで一括送信）

```
ありがとうございます。コアの 10 問が埋まったので、続けて詳細シート 20 問を送ります。
こちらも書ける範囲で OK、難しい項目は「わからない」「後で確認」と書いていただいて構いません。

【詳細シート（20 問）】

事業の現在地（追加）
Q11. 今、一番売りたい商品/プランは？
Q12. 直近 3 か月で一番うまく売れたお客さんは、どんな人/会社でしたか？

JTBD（追加）
Q13. お客さんが検討した他社さんはどこでしたか？
Q14. 検討中に一番不安そうだったポイントは？

ペイン / ゲイン（追加）
Q15. 他社さんと比べてよく褒められる点／negative に言われる点は？
Q16. お客さんがウチを誰かに紹介する時、どう説明していますか？

サイトの役割（追加）
Q17. どこから流入してくる人が多いですか？（広告/紹介/検索/SNS）
Q18. 既存サイトを見たお客さんからよく聞かれる質問は？

オファー（追加）
Q19. 保証・返金・無料体験はありますか？
Q20. 特典・キャンペーンは現在ありますか？
Q21. 競合と比べて価格はどのポジションですか？（安い / 中位 / 高い）

制約・前提
Q22. 公開希望日 / 納期と、絶対に外せない日付は？
Q23. 載せられない情報（NG 表現、未公開数値）は？
Q24. 表現 NG・参考にしてほしくないサイトはありますか？

競合・参考（追加）
Q25. 直接の競合だと思う会社（3 社以内、社名で）:
Q26. 「絶対こうはしたくない」サイトは？

ブランド（追加）
Q27. お客さんに与えたい印象は？（プロっぽい / 親しみやすい / 革新的 など）
Q28. NG な表現・トーンは？

クロージング
Q29. 今回のサイト制作で「これだけは絶対に成功させたい」というポイントは？
Q30. 何か言い忘れたこと、相談したいことはありますか？（自由記述）
```

回答後:
- 発話ログに記録
- coverage を再評価
- 主要 6 セルすべて埋まれば `hearing-report` skill 起動
- 不足セルが残れば `followup-session` で必要分だけ追質問

## ⑤ 完了通知

```
ヒアリング完了ありがとうございました。

ここから先は裏で進行します:
- まず要件整理エージェントが、いただいた回答と事前資料を要件定義書にまとめます
- 次にリサーチエージェントが、競合サイト・市場動向・VOC を並列で調査します
- 最後にブリーフエージェントが、後続の制作 AI エージェント向けの戦略ブリーフを生成します

進捗を見たい時は orchestrator に「<案件名> の進捗を教えて」と声をかけてください。
```

---

## NG / 注意

- シート送信は **必ず 1 メッセージ**。複数メッセージに分割しない（クライアントの心理負担が増える）
- クライアントが「全部書くのは大変」と言った場合は、コアシートだけ埋めてもらえば次に進める旨を伝える
- 全シート完了前に「待ちきれない」「もう要件整理して」と言われた場合は、その時点の埋まり具合で hearing-report に進む判断もあり（coverage を正直に書く）
