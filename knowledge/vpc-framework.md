# VPC Framework — Strategyzer Value Proposition Canvas

Customer Profile 側（Jobs / Pains / Gains）を徹底的に聞く。自社ソリューション側（Pain Relievers / Gain Creators / Products & Services）は **聞かない**（クライアントが既に知っている / brief 時点ではバイアスになる）。

## Customer Profile の 3 要素

### Jobs（顧客が片付けたい仕事）

顧客が「自分の人生 / 仕事の中で進めようとしていること」。3 層に分けて聞く：

- **機能的 Jobs**: 目的を達成するための具体タスク（例: 「広告制作の納期を守る」）
- **社会的 Jobs**: 他者からどう見られたいか（例: 「上司から評価されたい」）
- **感情的 Jobs**: どう感じたいか（例: 「焦りを減らしたい」「自信を持ちたい」）

質問例:
- 「お客さんが最近この商品を使った瞬間、その人は何を達成しようとしていましたか？」
- 「その時、上司や同僚にはどう思われたかったでしょうか？」

### Pains（顧客が嫌がっていること）

「Jobs を進める時に邪魔になっていること」「ネガティブな結果」「リスク」。

- **障害**: Jobs を遅らせる / 妨げる事柄
- **嫌な結果**: 起こったら困ること
- **リスク**: 起こりうる悪い未来

質問例:
- 「お客さんがよく口にする愚痴・困りごとは？」
- 「『これだけは避けたい』とお客さんが言うものは？」

### Gains（顧客が望んでいること）

「Jobs を進めた先で欲しい成果」「ポジティブな期待」「夢」。

- **必須 Gains**: 最低限欲しいもの（無いと不満）
- **期待 Gains**: あれば嬉しいもの（顧客がよく口にする）
- **望み Gains**: あったら最高（顧客は明言しないことも多い）
- **想定外 Gains**: 顧客が思いつかない感動

質問例:
- 「契約後に『これは助かった』と言われたことは？」
- 「『これがあったら最高なのに』とお客さんが愚痴ったことはある？」

## ヒアリングの順序

VPC を聞くお勧め順:

1. Jobs（機能的 → 社会的 → 感情的）
2. Pains（障害 → 嫌な結果 → リスク）
3. Gains（期待 → 必須 → 望み）

Jobs → Pains は自然な流れ。Gains は最後にすると「ない物ねだり」が出やすい。

## 生の言葉を残す

VPC は **顧客の生の言葉そのものをコピーに使う** ことが核心。意訳・要約しない。

例:
- ❌ 要約: 「制作の効率化を望んでいる」
- ✅ 生の言葉: 「金曜の夜に来週分の入稿が終わらなくて、毎週土曜潰れてる」

ヒアリングログに 生の引用を `[<client-name>] "..."` の形で残す。

## VPC 完了の基準

- Jobs: 機能的 / 社会的 / 感情的のいずれかで合計 5 つ以上の具体記述
- Pains: 障害 / 結果 / リスクで合計 5 つ以上
- Gains: 期待 / 必須 / 望みで合計 3 つ以上

これらが揃わない場合は `followup_questions` に追加。
