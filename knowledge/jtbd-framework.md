# JTBD Framework — Bob Moesta 式の運用

Jobs-To-Be-Done インタビューの目的は「顧客が何を雇うために商品を買ったか」を、過去の具体的行動から再構成すること。Bob Moesta の方法論を中小企業向けに圧縮した運用ルール。

## 起点質問

> 「契約してくれた最近のお客さんの中で、一番典型的だった人を1人思い浮かべてください。
> その人が、ウチを探し始めた時、どこで何をしていて、何で困っていましたか？」

仮定の未来（「どんな客が欲しい？」）ではなく **過去の具体的行動** を聞くのが鉄則。

## 4つの時間軸

Moesta は「first thought（初めて考えた瞬間）」から「purchase（購入）」までを4ステップで聞く：

1. **First Thought**: 初めて「これ何とかしないと」と思った瞬間
2. **Passive Looking**: 解決策を漫然と探し始めた時期
3. **Active Looking**: 具体的に比較検討した時期
4. **Decision**: 「これだ」と決めた瞬間

各ステップで「誰と話したか」「何を見たか」「何が引き金だったか」を聞く。

## 10 のコア質問（簡易版）

1. 解決策を探し始めたきっかけは？（First Thought のトリガー）
2. 何を達成したかったか？（機能的・感情的の両方）
3. 検討した代替案と、その理由
4. 決断の前にぶつかった障害は？
5. 問題解決のために取ったステップ
6. 期待した成果と成功の測り方
7. 既存のやり方での不満
8. 「これだ」と決めた瞬間（決定打）
9. どうなったら他に乗り換えるか
10. 理想の解決策はどんな姿か

中小経営者向けには質問数を絞り、`knowledge/question-bank.md` の B ブロック（5問）に圧縮済み。

## 抽象 → 具体への戻し方

クライアントが「便利」「効率的」「品質が良い」など抽象語を使ったら：

| 抽象語 | 戻す質問 |
|---|---|
| 便利 | 「具体的にどんな場面で？最後に同じことが起きたのはいつ？」 |
| 効率 | 「どれくらい時間 / コストが減りましたか？」 |
| 品質 | 「お客さんが『これは品質が良い』と言った具体的な瞬間は？」 |
| 信頼 | 「信頼してもらえた瞬間、お客さんが何を見て・何を聞いて納得しましたか？」 |
| 安心 | 「不安だったポイントが解消した瞬間、何が起きましたか？」 |

## 「決定打」の掘り方

質問 5「『これだ』と決めた瞬間」への深掘りシーケンス：

1. その瞬間、お客さんはどこにいましたか？
2. 何を見ていましたか？（資料・サイト・人）
3. 誰と一緒にいましたか？（社内・パートナー・家族）
4. 直前に何が起きましたか？（イベント・問い合わせ・他社との比較）
5. その瞬間、頭に浮かんだ言葉は？（あれば再現してもらう）

この5要素が揃うと、後続のコピー設計・ヒーローセクション設計に直接使える。

## ヒアリング中の禁則

- 「もし〇〇だったら？」「将来〇〇したいですか？」（仮定の未来）
- 「ペルソナは？」「ICP は？」（マーケ用語の直撃）
- 「USP は？」（事業語に言い換える）
- 答えを誘導する質問（「やっぱり〇〇ですよね？」）

## ヒアリング完了の基準

JTBD 観点での「最低限揃ったライン」：

- Trigger（解決策を探し始めたきっかけ）: 具体的な状況/イベントが1つ取れている
- Decision moment（決定打）: 具体的な瞬間が1つ取れている
- Competitive alternatives（検討した代替案）: 2つ以上挙がっている
- Pains（解消したかった困りごと）: 3つ以上挙がっている

これらが揃わない場合は `followup_questions` に追加し、次セッションで掘る。
